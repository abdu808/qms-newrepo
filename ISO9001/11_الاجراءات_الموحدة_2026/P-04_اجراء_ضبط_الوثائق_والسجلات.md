# P-04 — إجراء ضبط الوثائق والسجلات
**الإصدار:** 1.0 · **المالك:** مدير الجودة · **البند ISO:** 7.5

## 1. الغرض
ضمان توفر الوثيقة الصحيحة في المكان الصحيح للشخص الصحيح، ومنع استخدام الوثائق القديمة أو غير المعتمدة.

## 2. أنواع الوثائق
| النوع | الترميز | أمثلة |
|------|-------|-------|
| سياسة | POL | سياسة الجودة، سياسة المالية |
| إجراء | PROC | هذا المستند |
| تعليمات عمل | WI | كيفية إدخال تبرع |
| نموذج | FORM | طلب مستفيد |
| سجل | RECORD | سجل المستفيدين |
| خطة | PLAN | خطة التدقيق السنوية |
| تقرير | REPORT | تقرير الأداء الشهري |

## 3. دورة حياة الوثيقة
```
DRAFT (المُعِدّ) ──▶ UNDER_REVIEW (المراجع) ──▶ APPROVED (المعتمد) ──▶ PUBLISHED
                                                                          │
                                                          OBSOLETE ◀─────┘
```

## 4. الخطوات
### 4.1 الإنشاء
- المنشئ يحدد: العنوان، الفئة، الجمهور، المحتوى.
- يحفظ كـ **DRAFT** في شاشة `/documents`.
- يرفق النسخة الإلكترونية (PDF/DOCX) كـ DocVersion.

### 4.2 المراجعة
- يغيّر المنشئ الحالة إلى **UNDER_REVIEW**.
- مدير الجودة يراجع المحتوى خلال 5 أيام عمل.
- إن كان يحتاج تعديلاً: يرفض ويعيدها إلى DRAFT.

### 4.3 الاعتماد
- مدير الجودة يعتمد عبر زر "✅ اعتماد" → APPROVED.
- التاريخ الآلي مسجَّل.

### 4.4 النشر والإقرار
- تغيير الحالة إلى **PUBLISHED**.
- الجمهور المستهدف (audience) يُبلَّغ آلياً.
- كل موظف يقرأ ويُقرّ إلكترونياً (Ack) خلال 7 أيام.
- **لا يُسمح بالعمل بنسخة غير منشورة.**

### 4.5 المراجعة الدورية
- كل وثيقة لها `reviewDate` (تاريخ مراجعة).
- قبل 30 يوم من الانتهاء: تنبيه آلي لمدير الجودة.
- إن لم تُراجَع: تُحوَّل تلقائياً إلى OBSOLETE.

### 4.6 السحب والأرشفة
- وثيقة OBSOLETE تُحتفظ في النظام (read-only) لـ 7 سنوات.
- يُتاح فقط للمدققين الخارجيين والمدير الأعلى.

## 5. قواعد حاكمة
- **لا يُسمح بوثيقتين فعّالتين بنفس الرمز**.
- رقم الإصدار إلزامي (1.0, 1.1, 2.0 ...).
- أي تعديل جوهري يستلزم رقم إصدار major (1.x → 2.0).
- الموافقة الإلكترونية (Ack) إلزامية قبل العمل بالوثيقة.

## 6. السجلات
- Documents (كل الوثائق بكل حالاتها)
- DocVersions (النسخ المرفقة)
- Acks (إقرارات الموظفين)
- سجل التدقيق (AuditLog) لكل تغيير حالة

## 7. KPIs
- Q-08: نسبة الوثائق المعتمدة = **100%**
- Q-09: نسبة الوثائق المنتهية = **0%**
- نسبة الإقرار خلال 7 أيام = **100%**
