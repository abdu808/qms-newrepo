## 🔴 الأولوية القصوى — أخطاء حرجة

### A1 — حقن HTML في النماذج العامة (ثغرة أمنية)
**الملف:** `publicSurvey.js` + `publicEval.js`  
**المشكلة:** عناوين الأسئلة والملاحظات تُدرج مباشرةً في HTML دون تهريب  
**الخطر:** هجوم XSS على النماذج العامة (الاستبيان / تقييم المورد)  
**الإصلاح:** إضافة دالة `escapeHtml()` وتطبيقها على كل مدخل يُعرض في HTML  
- [x] إضافة `escapeHtml` في `publicSurvey.js`
- [x] إضافة `escapeHtml` في `publicEval.js`

---

### A2 — حقن فلاتر Prisma (ثغرة أمنية)
**الملف:** `crudFactory.js` سطر 39-42  
**المشكلة:** `req.query.filter` يُمرر مباشرةً لـ Prisma بدون قائمة بيضاء  
**الخطر:** تصفية بيانات غير مصرح بها (`?filter[password]=x`)  
**الإصلاح:** قبول فلاتر من `opts.allowedFilters` فقط  
- [x] تعديل `crudFactory.js` لقبول `allowedFilters` من كل راوتر
- [x] تعريف `allowedFilters` في كل راوتر (ncr, complaints, risks, audits, suppliers)

---

### A3 — عدد السياسات المُلغاة يبقى صفراً
**الملف:** `qualityPolicy.js` سطر 62-63  
**المشكلة:** `deactivatedPreviousCount: 0` مكتوب ثابتاً بدلاً من `deactivated.count`  
**الإصلاح:** سطر واحد  
- [x] تعديل السطر 63 في `qualityPolicy.js` (إعادة هيكلة المعاملة)

---

### A4 — حقل التحقق من الفعالية في NCR لا يُسجل تدقيقاً
**الملف:** `ncr.js`  
**المشكلة:** تغيير حالة NCR إلى CLOSED لا يُنشئ سجل تدقيق منفصل بحدث "VERIFY"  
**الإصلاح:** إضافة `auditLog.create` عند إغلاق NCR  
- [x] إضافة سجل تدقيق مستقل عند الإغلاق (VERIFY_NCR_EFFECTIVENESS)

## 🟠 الأولوية العالية — ثغرات ISO

### B1 — ضبط الموردين (ISO 8.4): المورد يُستخدم قبل اعتماده
**الملفات:** `suppliers.js` + `publicEval.js`  
**المشكلة:** لا يوجد ما يمنع رفع مورد لحالة APPROVED بدون تقييم ناجح مسبق  
**الإصلاح:**
- [x] إضافة فحص في `suppliers.js` عند تغيير الحالة إلى APPROVED: يجب وجود تقييم ناجح
- [x] رسالة خطأ واضحة بالعربية

---

### B2 — التوثيق (ISO 7.5): لا يوجد API لسجل الإصدارات
**الملف:** `documents.js`  
**المشكلة:** `DocVersion` موجود في الـ schema لكن لا يوجد endpoint لاسترجاعه  
**الإصلاح:**
- [x] إضافة `GET /api/documents/:id/versions` يعيد قائمة الإصدارات
- [x] إضافة زر "📋 الإصدارات" + مودال عرض في واجهة الوثائق

---

### B3 — الأهداف (ISO 6.2): نسبة التقدم بدون تحقق
**الملف:** `objectives.js` / `crudFactory.js`  
**المشكلة:** حقل `progress` يقبل أي رقم (حتى -10 أو 200)  
**الإصلاح:**
- [x] إضافة `beforeUpdate` في `objectives.js` يتحقق أن `0 <= progress <= 100`

---

### B4 — المخاطر (ISO 6.1): لا يمكن إغلاق مخاطرة بدون إثبات معالجة
**الملف:** `risks.js`  
**المشكلة:** الحالة تنتقل من IDENTIFIED مباشرةً لـ CLOSED بدون خطة معالجة أو إجراء مثبت  
**الإصلاح:**
- [x] إضافة guard: لا إغلاق المخاطرة بدون `treatment` موجود
- [x] التحقق من server-side أن probability و impact بين 1-5 (clamp)

---

### B5 — التدريب (ISO 7.2): لا رابط بين التدريب ومتطلبات الكفاءة
**الملف:** `training.js`  
**المشكلة:** التدريبات منفصلة عن جدول الكفاءات المطلوبة  
**الإصلاح:**
- [x] إضافة حقل `competenceTarget String?` في schema.prisma
- [x] عرض "الكفاءة المستهدفة" في نموذج التدريب (ISO 7.2)

---

### B6 — المراجعة الإدارية (ISO 9.3.2): لا يوجد حقل حضور الإدارة العليا
**الملف:** `managementReview.js` + schema  
**المشكلة:** ISO 9.3 يشترط حضور الإدارة العليا؛ لا يوجد حقل للتوثيق  
**الإصلاح:**
- [x] إضافة حقل `topManagementPresent Boolean?` في schema.prisma
- [x] إضافة guard في managementReview.js: لا COMPLETED بدون تأكيد الحضور
- [x] إضافة checkbox في نموذج المراجعة الإدارية + دعم `bool` في النموذج العام

---

### B7 — الشكاوى (ISO 9.1.2): لا تصعيد تلقائي للشكاوى المتأخرة
**الملف:** `complaints.js`  
**المشكلة:** شكوى مفتوحة منذ 30 يوماً لا تُرسل أي تنبيه  
**الإصلاح:**
- [x] إضافة `GET /api/complaints/overdue` يُعيد الشكاوى المفتوحة > 14 يوم مع عمرها
